## 当前页面规则

### 抓取任务页

- 上半部分是两个并排模块：
  - 左侧 `关键词列表`
  - 右侧 `关键词`
- `关键词` 模块只保留添加词条功能。
- `关键词列表` 模块只显示词条本身。
- 每个词条右上角有灰色小 `×`，点击直接删除，不需要二次确认。
- 关键词列表上方显示“近 1 个月抓取过的历史词条”，点击后可重新加入。
- 默认分组是 `全部`。
- 在 `全部` 视图下，点击 `生成分组` 会将当前列表中的词条整体打包成一个新分组。
- 新分组会显示在 `全部` 右侧。
- 分组标签右上角有灰色小 `×`。
- 删除分组需要二次确认。
- 删除分组时，不删除词条本身，只移除词条的 `group_name`。
- 进入某个分组后，右侧新增词条默认归入当前分组。
- Dashboard 顶部统计卡片已删除。
- 抓取任务列表中的“预览结果”功能已删除。

### 词条设置页

- 页面按“词条”查看历史，不再按单次 digest 批次为主。
- 顶部展示所有抓取过的词条。
- 点击词条后，下方展示该词条过去每一天的历史记录。
- 页面里的“高级设置”模块已删除。
- “打开原摘要”按钮已删除。
- “X 篇文章”计数展示已删除。
- 来源不集中显示在卡片顶部。
- 来源必须跟在每条 bullet / 小标题的末尾。
- 一条 bullet 可以带多个来源超链接。
- 搜索输入框已加 `type=search`、`autocomplete=off`、密码管理器忽略标记，用于尽量减少 Safari / 自动填充异常弹层。

## 当前抓取逻辑

- 抓取逻辑以“词条”为单位。
- 同一词条一天最多抓取一次。
- 如果当天已经抓过，再次触发时直接复用当天结果。
- 如果当天还没抓取，才执行新抓取。
- 汇总摘要时，即使某些词条当天没有新内容，也应展示当天已复用的内容，而不是空白。

## 当前摘要展示规则

- 摘要详情页按关键词独立卡片展示。
- 每个关键词卡片彼此独立，不和其他关键词混合。
- 每张卡片展示自己的抓取日期。
- 老数据复用时，仍然要展示对应内容。
- 抓取任务状态文案统一为：
  - 有结果：`已完成`
  - 无新内容：`已完成（无新内容）`

## 技术约定

- 前端：Next.js 14 App Router + Tailwind CSS
- 后端：FastAPI
- 数据库：PostgreSQL
- 任务队列：Celery + Redis
- 抓取：requests + BeautifulSoup + readability-lxml + feedparser
- LLM：OpenAI SDK 兼容接口

## 安全与基础约定

- LLM API Key 和 SMTP 密码加密存储，不明文返回。
- JWT 使用 httpOnly cookie。
- FastAPI 使用 asyncpg；Celery worker 使用 psycopg2。
- 数据库仍有部分 schema 变更未完全转成 Alembic migration，生产前需要补齐。
